    INTRODUCTION

    Differences between DAAD and DAAD-Ready

    DAAD

    DAAD is a system for creating conversational adventures for 8- and 16-bit machines, developed primarily by Tim Gilberts. It was based on what had previously been The Quill, PAWS, and SWAN, created under the direct commission of the now-defunct Spanish company Aventuras AD.

    Unlike The Quill or PAWS, DAAD was not developed directly on the target machine; it was designed to run from DOS and Atari ST.

    DAAD remained lost for many years after Aventuras AD closed, until it was recovered in the 2010s.

    At that time, the system was exactly as it had been left in 1992—with the tools of that era, for the computers of that era, and with instructions on how to create the disks for the various supported platforms based on the systems of that time. Years later, the instructions on how to create adventures for the different machines became obsolete, and it was no longer necessary to transfer games to each machine for testing, thanks to the existence of emulators.

    Each DAAD user started to develop their own methods for creating the disks of a game for platforms such as CPC, Commodore 64, Amiga, Atari ST, MSDOS, ZX Spectrum, MSX, or PCW, using different utilities to create disks and virtual tapes for testing the game on emulators. This was complicated for many people, who spent more time figuring out how to create the final floppy disk or tape rather than developing the adventure. For that reason, and with the emergence of Maluva, DAAD Ready was born.

    Maluva

    Maluva is an extension of DAAD that allows the replacement of the typical vector graphics of the era on 8-bit machines with more attractive digital graphics. However, there was a caveat: Maluva only works with systems that use floppy disks. That is why Maluva was used to create games for Amstrad CPC, Spectrum +3, Commodore 64, and MSX 1, but only on floppy disk.

    Maluva was also ported to enable the use of certain variants of the ZX Spectrum, specifically the ZX Spectrum Next, the ZX-Uno, and a ZX Spectrum with DivIDE or DivMMC, thereby also allowing digitized graphics for these machines.

    New Interpreters

    Later, new interpreters were developed, capable of running games made in DAAD on machines other than the originals, or on variations of the original machines:

    • MSX 2 (MSX2DAAD): built from scratch and allowed the use of MSX 2’s enhanced graphics among other features.
    • Commodore Plus/4: a fork of the Commodore 64 interpreter, enabling a version that also supported Maluva.
    • MSDOS (PCDAAD): although there was already a DAAD interpreter for DOS that only supported 16-color VGA mode, a new one was developed which supports both VGA and 256-color SVGA modes.
    • HTML (jDAAD): a port of the previous one that generates a web page using HTML and JavaScript, allowing 256-color graphics.
    • ZX Spectrum 128k (ZXDAAD128): capable of running vector games on a cassette tape game.
    • Windows and MacOS (ADP): although still experimental, they support the same graphics as the Amiga or Atari ST.

    There is also another interpreter capable of running a game designed for any of DAAD’s original platforms on any operating system that supports Python: NAPS. We will not cover NAPS in this document, as creating graphics for NAPS essentially involves creating games on a classic platform, meaning there is no difference between creating graphics for, say, CPC and creating them for NAPS running in CPC mode.

    DAAD Ready

    DAAD Ready is a set of utilities and scripts for Windows that allows authors to focus on creating adventures, taking on all the tedious work of creating a .D64 file for Commodore 64, or a DSK for Spectrum +3, an ADF for Amiga, etc.

    Thus, by simply editing a file that indicates what happens in the adventure and placing the necessary graphics in a specific folder in the format required for each platform, you just double-click a script— for example, C64.BAT— and it generates a .D64 floppy disk file (the common extension for disks in Commodore 64 emulators) ready for distribution. Moreover, it even opens that file in a Commodore 64 emulator for testing.

    Obviously, the graphics differ depending on the computer used. Therefore, if you are going to add graphics to your adventure, you must create different graphics for each machine. DAAD Ready helps you do that, but it does not do it for you.

    One of the simplifications of DAAD Ready, which—aside from making complicated tasks easier—was born from the idea of simplifying some of DAAD's complexities, is that it always follows the rule of placing the graphic at the top (in the first 96 scanlines of the screen) and the text at the bottom. This is because, on one hand, it is the most common layout in adventures, and on the other hand, because Maluva and some other functionalities discussed in this document require the graphic to be at the top of the screen.

    Of course, it is possible to place graphics in other parts of the screen, but perhaps not in digital form, or not on all machines, or not in the desired video mode. Placing a graphic of 96 pixels in height and the full width of the screen is always possible on all platforms, so DAAD Ready uses this as the "common denominator" for all systems.

    Graphics by Platform

    Adding graphics to adventures in DAAD Ready is simple in theory but laborious in practice. In principle, you just need to place the graphics in the 'images' folder, in the format appropriate for each platform, and name them with the location number using three digits. For example, for ZX Spectrum +3 the classic SCR format of 6912 bytes is used, so to add the graphic for location 1, you would simply place the file 001.SCR in the 'images' folder containing the location graphic.

    You can also create graphics that are not tied to a specific location; for instance, adding 200.SCR, which can later be displayed in DAAD using an instruction called PICTURE on demand.

    The general rule in DAAD Ready is that the graphics we provide are always full-screen, and DAAD Ready takes care of cropping the graphic to the usual height of 96 pixels and displaying it on screen.

    ZX Spectrum 48K

    This platform is the easiest to explain because DAAD Ready does not support graphics for the 48K Spectrum. A 48K Spectrum is very limited in RAM, making it nearly impossible to load digitized graphics; only a few could fit. Therefore, the only way to add graphics is to use the old method of drawing graphics (vector-based), which is beyond the scope of DAAD Ready.

    ZX Spectrum +3

    With the Spectrum +3, things change. The +3 has 128K of RAM and, more importantly, a floppy drive, which allows for the loading of digitized graphics on the fly, simply by reading them from the disk without the need to occupy memory.

    To add graphics, you only need to create SCR files. DAAD Ready uses the .SCR format of 6912 bytes and places them in the "images" folder of DAAD Ready. The file name must be the location number in three digits with the .SCR extension. For example, to add the graphic for location 1, simply place the file 001.SCR containing the location graphic in the 'images' folder.

    Remember that only the first 96 pixels will be displayed, so anything below that pixel in the image is irrelevant.

    A very useful tool for creating graphics in SCR format is ZX Paintbrush.

    If you have a digital image, you can also use DaDither, selecting ZX Spectrum as the platform and choosing the 256x192x4 option for Screen. Considering the limited graphic capabilities of the Spectrum, converting an image with DaDither is sometimes complicated, and as you may have seen in some adventures, graphics are usually in black and white or minimally colored. The Mode/Gray option in DaDither allows for this.

    Keep in mind that each image will take up slightly more than 3.3K on the disk, so there is a limited amount of graphics that can fit on a 3" Spectrum +3 floppy disk. Some authors have opted to place graphics on both sides and ask the user to flip the disk when moving past a certain area or returning to the initial zone.

    ZX Spectrum 128K

    The ZX Spectrum 128K is an exceptional case within DAAD Ready because it is the only supported platform that generates files for cassette tapes, specifically the well-known .TAP format. This system makes use of one of the new interpreters that is not part of the Original DAAD: ZXDAAD128.

    This interpreter utilizes the extra memory of the ZX Spectrum 128K, as well as image compression, to accommodate several digitized graphics in memory.

    Even so, the RAM available in a 128K is limited, and no one would want to load the tape continuously to see new graphics, so you have to measure how many graphics you’re going to include because not all of them might fit.

    A common solution is to have a generic graphic that can be used in several locations, or "filler" graphics that are not really graphics but rather a logo or similar, or simply to have locations without a graphic.

    Due to the type of compression used by ZXDAAD128, the more consecutive horizontal pixels of the same color there are in the graphic, the better it compresses, and the more graphics can fit.

    The same graphics used in Spectrum +3 will automatically load on the ZX Spectrum 128K, but keep in mind that dithering with DaDither is almost the worst scenario in terms of image compression, so it is not highly recommended unless you have very few graphics.

    Note: For those who may have doubts, when we refer to the ZX Spectrum 128K we mean both Sinclair’s original ZX Spectrum 128K and Amstrad’s +2 models. We are not referring to the Spectrum +3, which, although it is a 128K Spectrum, has its own section.
